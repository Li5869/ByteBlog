# PDF → Markdown 八股文转换

## 概述

将面试八股 PDF 文件批量转换为结构化 Markdown 文档。适用于小林 coding、面经合集等面试材料的格式化整理。

## 输入

| 参数 | 必填 | 说明 |
|------|------|------|
| `pdf_path` | ✅ | PDF 文件路径 |
| `reference_md` | ❌ | 参考格式模板的 MD 文件路径（如有） |
| `output_dir` | ✅ | 输出目录路径 |
| `split_by` | ❌ | 分类维度：`topic`（按面试题主题）/ `chapter`（按章节）/ `single`（不拆分），默认 `topic` |
| `remove_langs` | ❌ | 需要去除的语言内容，如 `["C++", "Go"]` |
| `encoding_fix` | ❌ | 是否自动修正繁体/异体字为简体，默认 `true` |

## 输出

- 按主题拆分的多个 `.md` 文件
- 统一格式：`## 面试题标题` + `### 回答（可直接背诵）` + `### 详细解答（深入理解）`

## 工作流程

```
Step 1: 环境准备
  ├─ 检查 PyPDF2 是否已安装，未安装则 pip install
  └─ 确认 PDF 文件存在且可读

Step 2: PDF 文本提取
  ├─ 使用 PyPDF2 逐页提取全文
  ├─ 统计总页数和文本长度
  └─ 输出提取摘要供用户确认

Step 3: 文本预处理
  ├─ 繁体/异体字 → 简体中文修正（如「⾯」→「面」、「⾼」→「高」）
  ├─ 去除与面试八股无关的内容（广告、目录、页码等）
  ├─ 去除指定语言的八股文（如 C++、Go）
  └─ 图片引用保留文字说明（如"[图片: xxx 架构图]"）

Step 4: 内容分析与分类
  ├─ 扫描章节标题，建立主题分类索引
  ├─ 按分类维度拆分内容块
  └─ 如有参考模板，对齐格式风格

Step 5: Markdown 生成
  ├─ 每个主题生成独立 MD 文件
  ├─ 文件命名：`{序号}-{主题名}八股文.md`
  ├─ 内容格式：
  │   ## 面试题标题
  │   ### 回答（可直接背诵）
  │   > 简洁的口语化总结，适合直接背诵
  │   ### 详细解答（深入理解）
  │   深入的技术原理分析...
  └─ 附录：来源说明、文件统计

Step 6: 验证与清理
  ├─ 检查输出文件完整性（行数、题目数）
  ├─ 如有参考模板，对比格式一致性
  └─ 输出转换报告
```

## 常见编码问题处理

PDF 提取的文本常有以下问题，需自动修正：

| 原始字符 | 修正为 | 说明 |
|----------|--------|------|
| ⾯ | 面 | 康熙部首 |
| ⾼ | 高 | 康熙部首 |
| 索→索 | 索 | 异体字 |
| 裏 | 裡/里 | 异体字 |
| 問 | 问 | 繁体 |

## 使用示例

```
用户: 帮我把 xxx.pdf 转为 Markdown 八股文，参考 yyy.md 的格式，输出到 output/ 目录
```

## 注意事项

- PyPDF2 对扫描版 PDF 支持较差，如遇乱码需提示用户
- 转换前先提取前 2 页让用户确认质量
- 大文件（>100 页）建议分批处理
- 图片内容无法提取，保留文字描述即可
