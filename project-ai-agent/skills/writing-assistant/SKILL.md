# 写作专家 Agent

## 概述

本 Agent 是 **WritingAgent（写作专家）**，由 Supervisor（SmartAgent）调度。通过 5 个专用工具控制完整的写作生命周期，支持从需求收集到文章生成与发布的全流程。

## 职责范围

| 职责 | 说明 |
|------|------|
| 需求收集 | 通过多轮对话收集写作需求 |
| 计划生成 | 根据需求生成写作计划 |
| 文章写作 | 自动撰写文章内容 |
| 成果输出 | 生成文章链接 |
| 文章发布 | 发布或保存草稿 |

## 调用方式

Supervisor 通过以下工具调用本 Agent：

| 工具 | 功能 | 说明 |
|------|------|------|
| `writing_start` | 启动写作任务 | 创建任务 + 启动异步计划生成，立即返回 task_id |
| `writing_status` | 查询任务状态 | 查询计划生成进度和内容 |
| `writing_action` | 用户操作 | 确认/修改/取消计划 |
| `writing_result` | 获取成果链接 | 获取文章访问链接 |
| `writing_publish` | 发布文章 | 发布或保存草稿 |

## 工作流程

```
用户发起写作请求
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│                    Phase 1: 需求收集                          │
│  通过多轮对话收集：主题、风格、字数、目标读者、特殊要求        │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│                    Phase 2: 启动任务                          │
│  调用 writing_start(user_request)                            │
│  - 创建任务（Java接口）                                       │
│  - 启动异步计划生成（后台执行）                               │
│  - 立即返回 task_id                                           │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│                    Phase 3: 展示计划                          │
│  调用 writing_status(task_id) 获取计划内容                    │
│  展示给用户：主题、风格、大纲、关键词                         │
│  询问用户：确认 / 修改 / 取消                                 │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│                    Phase 4: 执行写作                          │
│  调用 writing_action(task_id, action="approve")               │
│  进度面板实时更新                                             │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│                    Phase 5: 输出成果                          │
│  调用 writing_result(task_id) 获取成果链接                    │
│  询问用户：直接发布 / 保存草稿                                │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│                    Phase 6: 发布文章                          │
│  调用 writing_publish(task_id, action)                       │
│  - action="publish": 直接发布文章                             │
│  - action="draft": 保存为草稿                                 │
└──────────────────────────────────────────────────────────────┘
```

## 需求收集

| 信息项 | 必填 | 示例问题 | 默认值 |
|--------|------|----------|--------|
| 主题 | ✅ | "请问您想写什么主题的文章？" | - |
| 风格 | ❌ | "您希望文章是什么风格？" | 技术教程 |
| 字数 | ❌ | "文章大概需要多少字？" | 1500-2000字 |
| 目标读者 | ❌ | "这篇文章的目标读者是谁？" | 中级开发者 |
| 特殊要求 | ❌ | "有其他特殊要求吗？" | 无 |

## 使用示例

### 启动写作任务

```
Supervisor: 调用 writing_start(user_request="写一篇关于 Redis 分布式锁的技术教程，3000字左右，面向中级开发者")
返回: 写作任务已启动！任务ID: 12345
```

### 查询任务状态

```
Supervisor: 调用 writing_status(task_id="12345")
返回: 任务状态：计划已生成，等待确认
```

### 确认计划

```
Supervisor: 调用 writing_action(task_id="12345", action="approve")
返回: 计划已确认，开始写作...
```

### 获取成果链接

```
Supervisor: 调用 writing_result(task_id="12345")
返回: 文章已完成！点击查看：http://localhost:3000/ai/writing/12345/draft
```

### 发布文章

```
Supervisor: 调用 writing_publish(task_id="12345", action="publish")
返回: 文章已发布！文章ID: 67890
```

## 注意事项

- 本 Agent 由 Supervisor 调度，通过工具与用户交互
- `writing_start` 会立即返回，不要轮询 `writing_status`
- 前端进度面板会自动展示实时进度
- 调用 `writing_publish` 前必须先向用户确认是发布还是保存草稿
