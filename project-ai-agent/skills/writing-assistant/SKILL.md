# 智能写作助手 Skill

## 概述

本 Skill 将 WritingAgent 封装为 SmartAgent 的子Agent，提供完整的写作任务全流程能力。支持意图识别、交互式需求收集、计划生成、用户确认、自动写作、成果链接输出和文章发布。

## 触发条件

当用户输入涉及以下意图时，调用本 Skill：

- 表达写作意图（如"我想写一篇文章"、"帮我写..."）
- 请求创作内容（如"能不能帮我写一个教程"）
- 需要生成博客文章（如"帮我生成一篇技术博客"）
- 明确的写作任务请求

**触发关键词示例**：
- "我想写一篇..."
- "帮我写..."
- "能不能帮我创作..."
- "写一个关于...的文章"
- "帮我生成一篇..."

## 工具列表（5个）

本 Skill 使用 5 个工具控制 WritingAgent 的完整生命周期：

| 工具 | 功能 | 调用方式 |
|------|------|----------|
| `writing_start` | 创建任务 + 启动异步计划生成，立即返回 task_id | Java接口 + WritingExecutionService |
| `writing_status` | 查询任务状态；plan_ready 时返回完整计划内容 | Java接口 + Redis |
| `writing_action` | 用户操作（确认/修改/取消） | WritingExecutionService |
| `writing_result` | 获取成果链接 | Java接口 |
| `writing_publish` | 发布或保存文章草稿 | Java接口（获取草稿 + 发布文章） |

## 需求收集阶段

SmartAgent 通过多轮对话收集以下信息：

| 信息项 | 必填 | 示例问题 | 默认值 |
|--------|------|----------|--------|
| 主题 | ✅ | "请问您想写什么主题的文章？" | - |
| 风格 | ❌ | "您希望文章是什么风格？（技术教程/经验分享/深度分析...）" | 技术教程 |
| 字数 | ❌ | "文章大概需要多少字？" | 1500-2000字 |
| 目标读者 | ❌ | "这篇文章的目标读者是谁？" | 中级开发者 |
| 特殊要求 | ❌ | "有其他特殊要求吗？" | 无 |

**需求整合示例**：

```
主题：{topic}
风格：{style}
目标字数：{word_count}
目标读者：{target_audience}
特殊要求：{special_requirements}
```

## 工具详细说明

### 1. writing_start（启动写作任务）

**功能**：创建写作任务并启动异步计划生成，立即返回 task_id。

**调用时机**：收集完用户需求后调用。

**重要行为说明**：
- **此工具会立即返回，不会等待计划生成**
- **不要轮询 writing_status 查询进度**
- 前端进度面板会自动展示实时进度
- 计划生成完成后，用户会在对话中提及，届时再调用 writing_status 获取

**参数**：
- `user_request`: 用户的写作需求描述

**返回示例**：
```
写作任务已启动！

**任务ID**: `12345`

正在生成写作计划，请通过下方进度面板查看实时进度。
计划生成完成后我会通知您。
```

**Agent 回答模板**：
```
✅ 写作任务已启动！

📝 任务ID: `12345`

正在生成写作计划，请通过下方进度面板查看实时进度。
计划生成后我会通知您～
```

### 2. writing_status（查询任务状态）

**功能**：获取任务的当前状态。仅以下情况调用：
- 用户主动询问进度时
- 用户询问计划内容时（状态为 plan_ready 时返回完整计划）
- 需要确认任务是否已完成时

**调用时机**：
- 用户主动询问写作进度时
- 用户询问"计划是什么"、"大纲是什么"时

**不要轮询此工具**，前端进度面板会自动实时更新。

**参数**：
- `task_id`: 写作任务ID

**进行中状态返回示例**：
```
📋 任务 `12345` 正在进行中，请通过下方进度面板查看实时进度。
我会在任务完成后通知您。
```

**plan_ready 状态返回示例**：
```
📋 **任务状态**：计划已生成，等待确认

📄 **写作计划**

**主题**：Redis 分布式锁实战教程
**风格**：技术教程
**预估字数**：3000字
**目标读者**：中级开发者

**大纲**：
1. 分布式锁概述与应用场景
2. Redis 分布式锁基本实现（SETNX）
3. Redisson 框架进阶用法

---
请问您要：
1. ✅ **确认** - 确认计划，开始写作
2. ✏️ **修改** - 提出修改意见
3. ❌ **取消** - 取消任务
```

### 3. writing_action（用户操作）

**功能**：对写作任务执行用户操作（确认/修改/取消）。

**调用时机**：用户对计划做出决定时调用。

**参数**：
- `task_id`: 写作任务ID
- `action`: 操作类型
  - `"approve"`: 确认计划，开始写作
  - `"revise"`: 修改计划（需提供 feedback）
  - `"cancel"`: 取消任务
- `feedback`: 修改意见（仅 `action="revise"` 时需要）

**使用示例**：
```
用户: 确认计划，开始写作
AI: 调用 writing_action(task_id="12345", action="approve")

用户: 我想把大纲改一下，增加一个性能优化章节
AI: 调用 writing_action(task_id="12345", action="revise", feedback="增加性能优化章节")

用户: 取消任务
AI: 调用 writing_action(task_id="12345", action="cancel")
```

### 4. writing_result（获取成果链接）

**功能**：获取写作任务成果的访问链接。

**调用时机**：任务状态为 `finalized` 时调用。

**参数**：
- `task_id`: 写作任务ID

**返回示例**：
```
✅ 您的文章已完成！

🔗 **点击查看完整文章**：http://localhost:3000/ai/writing/12345/draft

您可以查看并编辑文章内容，或直接发布到博客。
```

### 5. writing_publish（发布文章）

**功能**：发布或保存写作任务生成的文章。

**调用时机**：用户确认要发布或保存文章时调用。**必须先向用户确认操作类型**。

**参数**：
- `task_id`: 写作任务ID
- `action`: 操作类型
  - `"publish"`: 直接发布文章（status=1）
  - `"draft"`: 保存为草稿（status=0）

**重要说明**：
- 调用前必须先向用户确认是发布还是保存草稿
- 任务状态必须为 `finalized`
- 会自动获取草稿数据并转换为文章格式

**使用示例**：
```
用户: 直接发布吧
AI: 调用 writing_publish(task_id="12345", action="publish")

用户: 先保存为草稿，我再看看
AI: 调用 writing_publish(task_id="12345", action="draft")
```

**返回示例（发布成功）**：
```
✅ **文章已发布！**

📝 **文章ID**: `67890`

🔗 **点击查看文章**：http://localhost:3000/article/67890

🎉 恭喜！您的文章已成功发布到博客！
```

**返回示例（保存草稿）**：
```
✅ **文章已保存为草稿！**

📝 **文章ID**: `67890`

🔗 **点击查看文章**：http://localhost:3000/article/67890

📝 文章已保存为草稿，您可以随时编辑和发布。
```

## 前端进度展示

前端进度面板覆盖写作任务的完整生命周期：

**计划阶段**：
- `writing_start` 工具调用成功后，系统通过 Chat SSE 推送 `writing_plan_started` 事件
- 前端自动弹出进度面板，显示"正在生成写作计划"
- 计划生成完成后，面板显示"计划已就绪，请在对话中确认"

**执行阶段**：
- `writing_action(action="approve")` 执行成功后，系统通过 Chat SSE 推送 `writing_task_started` 事件
- 面板切换到执行进度模式，实时显示（标题→摘要→正文→标签→质量评估）
- 完成后显示"查看文章"按钮

**进度面板功能**：
- 计划阶段和执行阶段全覆盖
- 支持折叠/展开
- Writing SSE 流实时更新进度

## 注意事项

1. **需求收集**：确保收集到主题信息后再创建任务，其他信息可使用默认值
2. **不要轮询**：`writing_start` 返回后，不要轮询 `writing_status` 查询进度。前端进度面板会自动实时更新。只需告知用户"请查看下方进度面板"
3. **计划展示**：用户询问计划内容时调用 `writing_status` 获取，完整展示给用户
4. **用户决策**：等待用户明确选择（确认/修改/取消）后再调用 `writing_action`
5. **任务ID**：`writing_start` 返回的 task_id 已包含在返回内容中，后续工具调用需使用此 ID
6. **成果链接**：写作完成后调用 `writing_result(task_id)` 获取成果链接
7. **发布确认**：调用 `writing_publish` 前必须先向用户确认是发布还是保存草稿，不要擅自发布

## 与其他 Skill 的关系

- **smart-chat**：当用户进行一般性对话时使用，写作请求会触发 writing-assistant
- **article-search**：写作过程中可能需要搜索相关文章作为参考
- **knowledge-qa**：写作过程中可能需要查询知识库获取背景知识
