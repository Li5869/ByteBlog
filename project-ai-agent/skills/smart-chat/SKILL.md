# 智能对话 Skill

## 概述

本 Skill 提供**智能对话助手**能力，基于 **Supervisor 多 Agent 架构**。Supervisor 负责分析用户请求并调度专业 Agent 完成任务，支持多轮对话、工具调用和长期记忆，是用户与博客平台交互的核心入口。

## 触发条件

当用户输入涉及以下意图时，调用本 Skill：

- 一般性对话和闲聊（如"你好"、"帮我个忙"）
- 综合性问题（如"这个博客平台有哪些功能？"）
- 无法明确归类到其他 Skill 的请求

## Agent 团队

### Supervisor（调度者）

SmartAgent 作为 Supervisor，负责：
- 分析用户意图
- 调度专业 Agent 完成任务
- 汇总结果并生成回答
- 管理用户记忆（召回/保存）

### Sub-Agents（专业 Agent）

| Agent | 职责 | 调用方式 |
|-------|------|---------|
| search_agent | 博客文章搜索、博主搜索、外部技术博客搜索、网页爬取、分类查询 | `search_agent(task="搜索任务描述")` |
| knowledge_agent | RAG 知识库检索，支持项目知识和面试知识分类查询 | `knowledge_agent(task="知识库查询描述")` |
| code_execution_agent | 代码执行，支持多种编程语言 | `code_execution_agent(task="代码执行描述")` |

### WritingAgent（写作专家）

通过 5 个专用工具控制完整的写作生命周期：
- `writing_start` — 启动写作任务（异步，立即返回）
- `writing_status` — 查询任务状态
- `writing_action` — 确认/修改/取消计划
- `writing_result` — 获取成果链接
- `writing_publish` — 发布或保存草稿

```

└─────────────────┬────────────────────────┘
                  │

┌──────────────────────────────────────────┐
│          2. 意图分析与调度                 │
│  - LLM 分析用户意图                       │

└─────────────────┬────────────────────────┘
                  │

┌──────────────────────────────────────────┐
│          3. 任务执行                       │

                  │
```

## 调度原则

| 用户意图 | 调度目标 |
|---------|---------|
| 搜索类（找文章、找博主、搜外部资源、查分类） | search_agent |
| 知识类（项目实现、技术原理、面试准备） | knowledge_agent |
| 代码执行类（运行代码、测试代码） | code_execution_agent |
| 写作类（写文章、生成标题、修改文章） | WritingAgent 工具 |
| 能力类（你能做什么、功能怎么用） | 直接调用 skill 工具 |
| 通用类（时间、用户信息） | 直接调用通用工具 |
| 记忆类（记住这个、我之前说过什么） | recall_memory / save_memory |
| 复杂请求 | 多个 Agent 协作，按顺序依次调用 |

## 长期记忆

系统支持跨会话的用户长期记忆，提供个性化服务：

### 记忆类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 语义记忆 | 用户的偏好、属性、知识背景 | "用户偏好 Java 技术栈" |
| 情节记忆 | 关键对话的摘要 | "上周讨论过 Spring AI RAG 方案" |
| 程序记忆 | 用户的交互习惯 | "用户喜欢简洁回答" |

### 记忆策略

| 策略 | 时机 | 说明 |
|------|------|------|
| 首轮自动召回 | 对话开始时 | 自动召回用户记忆，注入上下文 |
| 按需召回 | 对话过程中 | LLM 调用 `recall_memory` 工具 |
| 主动保存 | 对话过程中 | LLM 调用 `save_memory` 工具 |
| 后台提取 | 对话结束后 | XXL-Job 定时调度（兜底机制） |

### 使用场景

**召回记忆**：
- 用户询问历史（"我之前说过什么？"）
- 用户提到之前的内容（"上次我们讨论的..."）
- 需要个性化服务（根据用户偏好调整回答）

**保存记忆**：
- 用户明确要求记住（"记住这个"、"帮我记住"）
- 用户表达偏好（"我喜欢..."、"我不喜欢..."）
- 做出重要决策（"我决定用..."、"我选择..."）

## 使用示例

### 示例 1：普通对话

```
用户：你好，请问这个博客平台主要关注哪些技术领域？

AI：你好！让我查一下博客的分类情况...
[调用 search_agent]

根据查询结果，本博客平台主要关注以下技术领域：

📂 **分类**：后端开发、前端开发、AI、数据库、运维

你可以根据这些分类浏览感兴趣的内容！
```

### 示例 2：记忆召回

```
用户：我之前和你讨论过什么？

AI：让我回忆一下...
[调用 recall_memory(query="用户之前的讨论")]

根据我的记忆，我们之前讨论过：
- 你偏好 Java 技术栈，喜欢简洁的代码风格
- 上周我们讨论了 Spring AI RAG 方案，最终选择了向量检索+重排序模式

你还想了解什么？
```

### 示例 3：记忆保存

```
用户：记住，我喜欢用 DeepSeek V4 模型

AI：好的，我记住了！
[调用 save_memory(content="用户喜欢用 DeepSeek V4 模型")]

已保存你的偏好。下次对话时我会参考这个信息。
```

## 多轮对话管理

- 使用 Redis 存储对话历史，支持上下文记忆
- 默认保留最近 20 条消息作为上下文
- 支持会话摘要功能（消息数、剩余 TTL）
- 用户可随时清空对话历史

## 注意事项

- 对于明确的搜索类请求，优先调用 search_agent，而非依赖 LLM 自身知识
- 工具调用结果应整合后再回复用户，避免直接返回原始 JSON
- 多轮对话中需关注上下文连贯性，避免重复介绍已提供的信息
- 当用户提问不明确时，主动引导用户给出更具体的信息
- 对于重要信息，主动调用 save_memory 保存，确保跨会话记忆
