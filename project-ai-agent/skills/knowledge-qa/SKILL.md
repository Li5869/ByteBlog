# 知识库问答 Skill

## 概述

本 Skill 基于 **RAG（检索增强生成）** 技术，通过向量相似度检索博客知识库，结合 LLM 生成准确答案。适用于用户提出具体技术问题、需要从已有博文中获取信息或进行知识查询的场景。

## 触发条件

当用户输入涉及以下意图时，调用本 Skill：

- 提出技术问题（如"什么是 LangGraph？"、"Spring Boot 怎么配置拦截器？"）
- 需要从博客中获取知识（如"博客里有没有关于 JWT 认证的文章？"）
- 询问平台已有内容（如"咱们博客有讲微服务的文章吗？"）

## 使用到的工具

| 工具函数 | 用途 | 来源文件 |
|---------|------|---------|
| `search_knowledge_base(query, top_k=5)` | 基于 pgvector 向量相似度搜索知识库，返回相关文档片段 | `tools/vector_tool.py` |
| `search_articles_by_keyword(keyword, limit=5)` | 当向量检索结果不足时，使用关键词搜索作为补充 | `tools/article_tool.py` |

## 工作流程

```
用户问题
    │
    ▼
┌─────────────────────┐
│ 步骤1: 问题分析      │
│ - 提取核心问题       │
│ - 改写优化查询语句   │
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 步骤2: 向量检索      │
│ - search_knowledge   │
│   _base(query, k=5)  │
│ - 获取相关文档片段   │
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 步骤3: 结果评估      │
│ - 检查结果数量和质量 │
│ - 不足时补充关键词   │
│   搜索               │
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 步骤4: 生成回答      │
│ - LLM 基于上下文     │
│   生成答案           │
│ - 标注信息来源       │
└─────────┬───────────┘
          │
       返回答案
```

## 详细说明

### 步骤1：问题分析

分析用户问题，提取核心查询意图，必要时对查询语句进行改写优化，提升向量检索的匹配效果。

### 步骤2：向量检索

调用 `search_knowledge_base(query, top_k=5)` 执行向量相似度搜索。内部使用 Parent-Child RAG 策略：

1. 使用 OpenAI Embedding 将查询文本向量化
2. 在 pgvector 中执行相似度搜索，匹配最相关的 Child Chunks
3. 通过 doc_id 关联到完整的 Parent Document
4. 返回完整的上下文内容

### 步骤3：结果评估与补充

- 若检索结果 ≥ 3 条且相关性良好 → 直接进入生成阶段
- 若检索结果 < 3 条 → 补充调用 `search_articles_by_keyword(keyword)` 获取更多上下文
- 若完全无结果 → 诚实告知用户知识库中暂无相关信息

### 步骤4：生成回答

基于检索到的文档片段，LLM 生成回答时需遵守：
1. 严格基于检索结果，不编造不存在的信息
2. 内容不足时如实告知，不强行回答
3. 在回答中标注信息来源（文章来源标题）
4. 保持专业性和准确性

## 使用示例

```
用户：什么是 LangGraph？博客里有相关介绍吗？
AI：好的，让我在知识库中检索相关信息...
[调用 search_knowledge_base("LangGraph", top_k=5)]
[找到 3 条相关文档]
根据博客文章，以下是关于 LangGraph 的介绍：

**LangGraph** 是 LangChain 团队推出的图编排运行时引擎，
用于构建底层工作流与 Agent。它提供：

1. **状态管理**：使用 TypedDict 定义状态，支持消息累积
2. **图结构编排**：通过 StateGraph 构建节点和边
3. **持久化运行**：支持 checkpoint 和中断恢复

> 📖 来源：《LangChain 生态架构解析》- 作者张三
```

## 注意事项

- 向量检索结果按相似度降序排列
- 当置信度较低时，应在回答中说明"仅供参考"
- 严格遵循 RAG 原则：不编造信息，不超出检索上下文作答
- 每次回答必须标注信息来源（文章标题），增强可信度
- 若用户连续追问，可结合对话历史中的检索结果进行回答
