# 知识库问答 Skill

## 概述

本 Skill 基于 **RAG（检索增强生成）** 技术，通过向量相似度检索知识库，结合 LLM 生成准确答案。支持根据问题类型自动路由到对应的知识库分类，实现精准检索。

## 知识库分类

| 分类 | ID | 内容范围 | 典型问题 |
|------|-----|---------|---------|
| 项目知识库 | `project` | 项目实现细节、系统架构、代码逻辑、部署配置 | "我们项目的认证怎么实现的？"、"系统的消息队列如何配置？" |
| 面试知识库 | `interview` | 技术原理、底层机制、面试题、最佳实践 | "Spring Bean 生命周期是什么？"、"Redis 有哪些持久化策略？" |
| 通用知识库 | `general` | 通用技术知识、学习资料 | "什么是微服务？"、"Docker 基础命令有哪些？" |

## 触发条件

当用户输入涉及以下意图时，调用本 Skill：

- 提出技术问题（如"什么是 LangGraph？"、"Spring Boot 怎么配置拦截器？"）
- 询问项目实现（如"我们系统的认证流程是怎样的？"）
- 面试相关问题（如"面试常问的 Redis 问题有哪些？"）
- 需要从知识库中获取知识（如"知识库里有没有关于 JWT 认证的内容？"）

## 使用到的工具

| 工具函数 | 用途 | 来源文件 |
|---------|------|---------|
| `search_knowledge_base(query, category=None, top_k=5)` | 基于 pgvector 向量相似度搜索知识库，支持按 category 过滤 | `tools/vector_tool.py` |
| `search_articles_by_keyword(keyword, limit=5)` | 当向量检索结果不足时，使用关键词搜索博客文章作为补充 | `tools/article_tool.py` |

## 工作流程

```
用户问题
    │
    ▼
┌─────────────────────────────┐
│ 步骤1: 意图识别与路由        │
│ - 分析问题属于哪个知识库     │
│ - 确定检索的 category 参数   │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 步骤2: 向量检索              │
│ - search_knowledge_base(     │
│     query,                   │
│     category=路由结果        │
│   )                          │
│ - 获取相关文档片段           │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 步骤3: 结果评估              │
│ - 检查结果数量和质量         │
│ - 不足时补充关键词搜索       │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 步骤4: 生成回答              │
│ - LLM 基于上下文生成答案     │
│ - 标注信息来源和知识库类型   │
└─────────────────────────────┘
```

## 详细说明

### 步骤1：意图识别与路由

根据用户问题内容，自动判断应该检索哪个知识库：

**路由规则**：

| 问题特征 | 路由到 | category 参数 |
|---------|--------|--------------|
| 包含"项目"、"系统"、"我们"、"实现"、"代码"、"模块"、"接口"、"配置"、"部署" | 项目知识库 | `project` |
| 包含"面试"、"原理"、"生命周期"、"底层"、"源码"、"区别"、"比较"、"什么是"、"如何理解" | 面试知识库 | `interview` |
| 无法判断或通用技术问题 | 全库搜索 | `None` |

**示例**：
```
用户："我们项目的认证流程是怎么实现的？"
路由：category="project"（包含"项目"、"实现"）

用户："Spring Bean 的生命周期是什么？"
路由：category="interview"（包含"生命周期"、"是什么"）

用户："Redis 怎么用？"
路由：category=None（无法判断，全库搜索）
```

### 步骤2：向量检索

调用 `search_knowledge_base(query, category, top_k=5)` 执行向量相似度搜索：

```python
# 项目相关问题
search_knowledge_base("认证流程实现", category="project", top_k=5)

# 面试相关问题
search_knowledge_base("Spring Bean 生命周期", category="interview", top_k=5)

# 通用问题（全库搜索）
search_knowledge_base("Redis 使用", category=None, top_k=5)
```

内部使用 Parent-Child RAG 策略：
1. 使用 Embedding 将查询文本向量化
2. 在 pgvector 中执行相似度搜索，按 category 过滤
3. 通过 doc_id 关联到完整的 Parent Document
4. 返回完整的上下文内容

### 步骤3：结果评估与补充

- 若检索结果 ≥ 3 条且相关性良好 → 直接进入生成阶段
- 若检索结果 < 3 条 → 补充调用 `search_articles_by_keyword(keyword)` 获取更多上下文
- 若完全无结果 → 诚实告知用户知识库中暂无相关信息

### 步骤4：生成回答

基于检索到的文档片段，LLM 生成回答时需遵守：
1. 严格基于检索结果，不编造不存在的信息
2. 内容不足时如实告知，不强行回答
3. 在回答中标注信息来源（文档标题）和知识库类型
4. 保持专业性和准确性

## 使用示例

### 示例1：项目知识库查询

```
用户：我们项目的消息队列是怎么配置的？
AI：让我在项目知识库中检索相关信息...
[调用 search_knowledge_base("消息队列配置", category="project", top_k=5)]
[找到 3 条相关文档]

根据项目知识库，消息队列的配置如下：

**RabbitMQ 配置**：
- 主机地址：配置在 application.yml 中
- 交换机类型：Direct Exchange
- 队列数量：15 个业务队列 + 6 个死信队列

> 📖 来源：项目知识库 - 《系统架构文档》
```

### 示例2：面试知识库查询

```
用户：Spring Bean 的生命周期是什么？
AI：让我在面试知识库中查找相关内容...
[调用 search_knowledge_base("Spring Bean 生命周期", category="interview", top_k=5)]
[找到 4 条相关文档]

**Spring Bean 生命周期**主要包含以下阶段：

1. **实例化**：创建 Bean 实例
2. **属性赋值**：注入依赖
3. **初始化**：执行初始化方法
4. **使用**：Bean 正常工作
5. **销毁**：执行销毁方法

> 📖 来源：面试知识库 - 《Spring 核心原理》
```

### 示例3：全库搜索

```
用户：Redis 有哪些持久化策略？
AI：让我在知识库中检索相关内容...
[调用 search_knowledge_base("Redis 持久化策略", category=None, top_k=5)]
[找到 2 条相关文档]

**Redis 持久化策略**主要有两种：

1. **RDB（快照）**：定时将内存数据保存到磁盘
2. **AOF（追加文件）**：记录每个写操作

> 📖 来源：面试知识库 - 《Redis 深度解析》
```

## 注意事项

- 向量检索结果按相似度降序排列
- 当置信度较低时，应在回答中说明"仅供参考"
- 严格遵循 RAG 原则：不编造信息，不超出检索上下文作答
- 每次回答必须标注信息来源和知识库类型，增强可信度
- 若用户连续追问，可结合对话历史中的检索结果进行回答
- 项目相关问题时优先使用 `project` 分类，避免与通用知识混淆
